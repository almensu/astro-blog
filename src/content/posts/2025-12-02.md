Hello 朋友们，见信如晤。

这周想和大家聊聊我刚刚上线的新博客。

为什么要折腾这个？除了想给自己搭个窝，还有一个特别私人的原因：**我实在受够了“为了发布而发布”的高摩擦系数。**

大家都有过这种时刻吧？心里刚冒出一个绝妙的观点，兴冲冲打开写作后台，结果第一眼看到的不是输入框，而是——**“请上传精美封面图（建议尺寸 900x383）”**。

那一瞬间，心里那股刚升起来的“气”，呲溜一下就被压下去了。找图、裁图、调色……这一套搞完，原本想表达的欲望也被磨没了。

**所以我一直是个坚定的“命令行党”。**

对我来说，写作就该是纯粹的。打开终端，写完文字，`git push`，搞定。没有封面图焦虑，没有排版黑洞，**这种“丝滑”感，才配得上我们珍贵的灵感。**

其实，搭建这个博客，也是我正在构建的\*\*「社区」\*\*版图中的一小块拼图。在这个拼图中，未来会有 Newsletter、有播客、有产品，而博客是那个沉淀思考的基石。

如果放在以前，想达成这种“丝滑”的代价可不小。漫长的选型、枯燥的技术文档、无穷无尽的 CSS 调试……对于执行力稍微弱一点的人（比如过去的我）来说，查文档查到一半，热情也就凉了，“歇菜”是常态。

**但这次完全不同，因为有了 AI。**

我发现，现在我们和“做成一件事”之间的距离，已经被 AI 无限缩短了。起心动念，就可以立即执行。不懂代码？让 AI 写。看不懂报错？让 AI 查。配置太繁琐？让 AI 生成。

**不过，这里我不想忽悠大家，必须诚实地说一句：编程基础依然是有用的。**

这并不是说你要精通每一行代码，而是**你的“技术直觉”决定了你能否写出精准的提示词**。你知道什么是“组件”、什么是“路由”、什么是“部署”，你才能指挥得动 AI 这个超级实习生。

**曾经，技术基础是拦住你的高墙；现在，它是你驾驭 AI 的方向盘。**

有了这点基础，再加上 AI 的执行力，我也终于把这块“拼图”一步步拼好了。这一篇，我想把这个过程完整复盘给你：**我是如何用 Astro + GitHub Pages + Cloudflare，在 AI 的辅助下，零成本搭起这个看着还不错的博客的。**

### 一、先想清楚“我到底要什么”

在动手之前，我先在纸上列了我的“硬性条件”，确保不为了折腾而折腾：

1.  **0 成本**：拒绝每年几百块的服务器费用，绝不为了“写字”这件事背负持续的财务成本。
2.  **Markdown**：我只习惯用 Markdown 写作，发布流程必须顺滑。
3.  **自定义域名**：这是博客的门面，不能用别人的二级域名。

其实市面上的托管方案很多。自建服务器太累，直接 Pass；Vercel 和 Cloudflare Pages 其实都很棒，但我最后还是选了 **GitHub Pages**。原因很简单：它跟代码天然绑定，作为程序员，把文章作为代码的一部分去 Commit，这种感觉最踏实，也最稳定。

至于技术栈，我放弃了之前用过的 Hugo 和 Hexo，转投了最近很火的 **Astro**。它的理念特别打动我：**内容优先（Content Focused）**，生成的全是静态 HTML，加载速度快到飞起。

### 二、从“纠结”到“拍板”：锁定 astro-gyoza

挑主题绝对是搭建博客过程中最杀时间的一环。

我也纠结过：是选那种设计感爆棚的？还是选那种朴素到像记事本的？最后我找到了一个平衡点——**`astro-gyoza`**。

这个主题（GitHub 地址：`lxchapu/astro-gyoza`）完美击中了我的好球区：

- **美观**：日系简约风，看着很舒服。
- **有 Hero**：首页很有设计感。
- **动效适中**：一点点微交互让页面有“生命力”，但不会抢走文字的风头。

一句话总结：它看起来“有设计感”，但大家都知道，内容才是主角。

### 三、把别人的“房子”搬回家

选好主题，接下来就是把它拉到本地。

`git clone ...` 这种命令就不多说了。我想重点带你看一眼它的结构，不要盲改，先看懂“房子”是怎么搭的：

```text
├── public/          # 放一些静态资源
├── src/
│   ├── components/  # 各种小组件（比如导航栏、页脚）
│   ├── content/     # 这里最重要，你的文章都在这
│   ├── layouts/     # 决定页面长什么样的模具
│   ├── pages/       # 决定你的网站有哪些页面
│   └── config.json  # 这是一个好东西，全站配置都在这
├── astro.config.mjs
└── ...
```

简单理解就是：以后你要写文章，就往 `content` 里扔 Markdown；你想改全站标题，就去动 `config.json`。

### 四、把模板改成“我的空间”

拿到代码的第一件事，就是把“样板间”改成“我的家”。

对于 `astro-gyoza` 这个主题，作者很贴心地把大部分配置提取到了 `src/config.json` 里。你要改的其实不多：

- 把 **Title** 改成你的博客名。
- 把 **Description** 换成你的 Slogan。
- 把 **Social Links** 换成你的 GitHub、X 或者邮箱。
- 去 `public` 文件夹替换掉原来的头像图片。

这里的经验是：**不要试图一口气把所有 UI 都改了**。先把最显眼的“门脸”改掉，看着像那么回事了，你的动力会更足。

### 五、开始写作：搞定关键的“元数据”

搞定了装修，就可以开始写第一篇文章了。

在 `src/content/posts` 目录下新建一个 `.md` 文件。这时候你可能会懵：除了写正文，头部那一坨代码是干嘛的？

那个叫 `frontmatter`（元数据区），是给博客系统看的“身份证”。它决定了你的文章标题、发布时间以及在列表页显示的摘要。

直接照着下面这个格式写就行：

```yaml
---
title: 从烦恼到顿悟：一个文本处理工具带来的复杂性思考
date: 2024-01-10
summary: 本文讲述了一个技术写作者如何通过构建自动化文本处理工具来解决专业术语记忆问题，并从中获得关于复杂性、简单性和人机协作的深刻思考。文章探讨了完美自动化的迷思、工具设计的核心原则，以及如何通过简单规则的组合构建优雅的解决方案。
category: 技术思考
tags: [Tools, 自动化]
---
```

**这里有个 AI 的超实用场景：**

有时候写完几千字的长文，真的脑子枯竭，完全不想写摘要（Summary），或者纠结该打什么标签（Tags）。

这时候，直接把写好的正文丢给 AI，说一句：

> “根据上下文，帮我生成这段 frontmatter，摘要写得吸引人一点，标签要精准。”

它生成的摘要往往比你自己憋半天写的还通顺。你只管负责核心创作，这种整理归纳的体力活，放心交给它。

### 六、本地测试：先在自己机器上跑通

改得差不多了，激动人心的时刻来了。在终端里敲下：

```bash
pnpm i    # 安装依赖
pnpm dev  # 启动本地服务
```

打开浏览器，访问 `localhost:4321`。

当看到带有我头像和 Slogan 的页面出现时，那种成就感已经完成了一半。这时候如果有报错（肯定会有），直接把报错信息完整复制给 AI，让它按目录结构带你排查。别自己死磕，AI 查 bug 比你快得多。

### 七、域名与 Cloudflare：让博客有个像样的门牌号

虽然 GitHub Pages 提供了免费域名，但我强烈建议你花点小钱买个顶级域名。比如我买了 `yanghoo205.com`，这不仅是网址，更是个人资产。

关于域名注册和解析，这里有个\*\*“经验之谈”\*\*：

- **如果是成熟的国内商业项目**：求稳、求国内访问速度，那在阿里云/腾讯云注册并老老实实备案是正道。
- **但对于我们这种刚起步的“初创”项目**：**Cloudflare**（或者配合 NameSilo/GoDaddy）才是首选。
  - **好处嘛，懂的都懂**：省去了漫长的备案流程，即买即用，最快速度验证想法。而且 Cloudflare 还免费送全球 CDN、HTTPS 证书和隐私保护。

在 Cloudflare 里，我添加了一条 CNAME 记录，把 `blog.yanghoo205.com` 指向了 GitHub Pages 的默认域名（稍后会用到）。

### 八、把部署交给 AI：GitHub Pages 上线

到了这一步，很多人会卡在“怎么配置 GitHub Actions”上。

别慌，继续用 AI。我直接对 AI 说：

> “我现在有个 Astro 项目，源码在 main 分支，构建输出目录是 dist。我想用 GitHub Actions 把它自动部署到 GitHub Pages，并且绑定自定义域名 blog.yanghoo205.com，请帮我写一份 workflow 配置文件。”

AI 会直接给你一份 `.github/workflows/deploy.yml` 文件。你只需要把它丢进项目里，推送到 GitHub。

然后在仓库设置里，把 Pages 的来源选为 `gh-pages` 分支（或者 Actions 自动生成的分支）。

稍等几分钟，看着 Actions 的绿色对号亮起。打开 `https://blog.yanghoo205.com`。

**Boom！上线了！** 🥳 给自己鼓个掌吧。
